## 문제 개요

길이가 `N`인 수식이 주어지며, 수식은 한 자리 정수(0~9)와 연산자 `+`, `-`, `×`로 구성된다.
모든 연산자의 우선순위는 동일하므로, 기본 계산은 왼쪽부터 순차적으로 수행된다.

수식에 괄호를 추가할 수 있으며:
- 괄호 안에는 **연산자 하나만 포함**
- 중첩된 괄호는 불가
- 괄호 개수 제한 없음 (0개 가능)

이때, 괄호를 적절히 추가하여 만들 수 있는 **수식 결과의 최댓값**을 구한다.

## 접근 방식 요약

이 문제는 괄호를 어디에 추가할지에 대한 모든 경우의 수를 탐색해야 한다.
연산 순서는 바꿀 수 없고 괄호는 특정 연산 하나를 먼저 계산하게 만드는 역할만 하므로,
각 연산자 위치에서의 선택을 **DFS**로 탐색한다.

## 시간 복잡도

$O(2^(N/2)) ≈ O(2^9)$

## 핵심 아이디어
1. 수식 분리
문자열 수식을 다음 두 배열로 분리한다.
- `nums[]` : 숫자 배열
- `ops[]` : 연산자 배열

예:
```
3+8×7-9
nums = [3, 8, 7, 9]
ops  = ['+', '×', '-']
```

2. 종료 조건
모든 연산자를 처리한 경우:
```java
if (idx == ops.length)
// 결과 최댓값 갱신
```
3. DFS 분기 로직
(1) 괄호를 사용하지 않는 경우 (항상 가능)

현재 연산자를 바로 계산한 후 DFS 호출한다.

(2) 괄호를 사용하는 경우 (조건부)

`idx + 1 < ops.length` 일때만

`(nums[idx+1] ops[idx+1] nums[idx+2])` 를 먼저 계산
그 결과를 `currentValue`와 `ops[idx]`로 계산


4. calc 함수
`calc(a, op, b)`는 단순히 하나의 연산을 수행하는 보조 함수이다.

`+` → `a + b`
`-` → `a - b`
`×` → `a * b`
